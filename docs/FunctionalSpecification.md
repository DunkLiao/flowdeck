# 功能規格書 (Functional Specification)

本文件詳細規範看板系統各模組的行為與預期輸出入。

## 1. 看板模組 (Board Module)

### 1.1 建立看板 (Create Board)
- **進入點**：左側側邊欄「新增看板」按鈕。
- **流程**：點擊後彈出單欄輸入框，使用者輸入看板名稱後提交。
- **限制**：
  - 看板名稱長度為 1-50 字元。
  - 不得空白或僅含空格。
- **預設行為**：新建的看板預設會建立三個預設欄位：「待處理」、「進行中」、「已完成」。

### 1.2 看板重命名 (Rename Board)
- **進入點**：點擊頂部導航列顯示的看板名稱。
- **流程**：名稱變為編輯狀態輸入框，失去焦點 (blur) 或按下 Enter 時儲存新名稱。
- **限制**：與建立看板相同。

### 1.3 刪除看板 (Delete Board)
- **進入點**：側邊欄看板項目右側的刪除按鈕。
- **流程**：彈出確認對話框，顯示「您確定要刪除看板『[看板名稱]』嗎？此動作將刪除該看板內的所有欄位與卡片，且無法還原。」
- **選項**：提供「確認刪除（紅色）」與「取消」按鈕。

### 1.4 看板切換 (Switch Board)
- **流程**：點擊側邊欄任一看板名稱，主工作區立即重新渲染對應的欄位與卡片，並將當前選定的看板 ID 記錄於 LocalStorage 以便下次重開網頁時恢復。

---

## 2. 欄位模組 (Column Module)

### 2.1 建立欄位 (Create Column)
- **進入點**：看板最右側的「+ 新增欄位」按鈕。
- **流程**：點擊後出現輸入框，輸入後新增，並將新欄位排在最後。

### 2.2 欄位重命名 (Rename Column)
- **進入點**：雙擊欄位標題。
- **流程**：標題變為輸入框，輸入後按 Enter 或失去焦點儲存。

### 2.3 刪除欄位 (Delete Column)
- **進入點**：欄位標題右側的操作選單 -> 刪除欄位。
- **流程**：彈出確認對話框。若欄位內有卡片，提示「此欄位內含有卡片，刪除欄位將一併刪除卡片。確定刪除？」

### 2.4 欄位拖曳排序 (Drag Reorder)
- **流程**：滑鼠拖曳欄位標題，可將欄位移動到看板中其他位置。拖曳時提供虛線預覽框，放開後更新排序屬性 (order) 並寫入資料庫。

---

## 3. 卡片模組 (Card Module)

### 3.1 建立卡片 (Create Card)
- **進入點**：每個欄位底部的「+ 新增卡片」按鈕。
- **流程**：點擊後在欄位最下方出現簡化輸入框，輸入卡片標題後 Enter 即可建立。
- **欄位預設值**：
  - `priority` 預設為「無」（或「低」）
  - `dueDate` 預設為空

### 3.2 編輯卡片 (Edit Card - Modal)
- **流程**：點擊卡片本體開啟 Modal，可修改：
  - **標題**：點擊即可編輯。
  - **描述**：提供 Markdown 編輯區域，支援「編輯模式」與「預覽模式」切換，或採用即時 Markdown 解析。
  - **優先權**：下拉選單提供「高」、「中」、「低」三種選項。
  - **到期日**：使用日期選擇器 (DatePicker)，支援清除到期日。
  - **標籤**：複選框，可為卡片附加/解除附加標籤，支援即時新增自訂顏色標籤。
  - **待辦清單 (Checklist)**：點擊新增項目，並可勾選，上方顯示進度條（如 `3/5 項目已完成`）。

### 3.3 卡片拖曳 (Drag & Drop)
- **流程**：卡片可在同欄位內上下拖曳排序，或拖曳至其他欄位。放開時更新卡片的 `columnId` 及 `order`。

---

## 4. 搜尋與篩選模組 (Search & Filter Module)

### 4.1 關鍵字模糊搜尋
- **欄位**：頂部搜尋框。
- **行為**：輸入內容時，對卡片標題 (title) 與描述 (description) 進行不分大小寫的字串匹配，不符條件者將隱藏 (display: none)。

### 4.2 屬性篩選 (Filter)
- **標籤篩選**：點擊標籤按鈕，可多選標籤。篩選方式為「聯集」（含有任何一個被選中的標籤即可）或「交集」（必須含有所有被選中的標籤）。
- **優先權篩選**：可勾選高、中、低。
- **到期日篩選**：可選取「已過期」、「今天到期」、「本週到期」。

---

## 5. 資料匯入/匯出與備份 (Data I/O & Backup)

### 5.1 JSON 匯出
- **行為**：點擊「匯出資料」後，打包資料庫內 Board, Column, Card, Tag, CardTag, Checklist 的所有紀錄，生成一個名為 `kanban_backup_YYYYMMDD_HHMMSS.json` 的檔案並觸發瀏覽器下載。

### 5.2 JSON 匯入
- **行為**：點擊「匯入資料」開啟檔案選取器。選擇 JSON 檔案後，進行欄位校驗。若格式正確，清除當前 IndexedDB 並覆寫，隨後強制重新整理頁面。

### 5.3 自動備份 (Auto-backup)
- **行為**：每一次寫入 IndexedDB 成功後，非同步將整份資料庫轉化為 JSON 字串存入 LocalStorage 鍵值 `KanbanDB_AutoBackup`。系統初始化時若發現 IndexedDB 空白但 LocalStorage 有此鍵值，則提示用戶是否從自動備份中回復。

### 5.4 還原初始設定
- **進入點**：左側側邊欄資料操作區的「還原初始設定」按鈕。
- **流程**：點擊後顯示危險操作確認對話框，提醒使用者目前所有看板、欄位、卡片、標籤與待辦清單都會被刪除，且自動備份會被覆寫。
- **確認後行為**：清空 IndexedDB 中 `boards`, `columns`, `cards`, `tags`, `cardTags`, `checklists` 全部資料，重建一個「我的工作看板」與三個預設欄位：「待處理」、「進行中」、「已完成」。
- **狀態更新**：將目前看板 ID 設為新建立的初始看板，覆寫 `KanbanDB_AutoBackup` 為初始資料狀態，顯示成功 Toast 後重新載入頁面。
- **取消行為**：若使用者在確認對話框按下取消或 Esc，資料庫與自動備份皆不變。
